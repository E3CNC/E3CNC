## CI integration

On tag push (`v*`), the Create Release workflow runs:

```bash
git-cliff --bump --output CHANGELOG.md
git add CHANGELOG.md
git commit -m "chore: update CHANGELOG for $(git describe --tags)"
git push
```

This keeps CHANGELOG.md current without requiring manual entries. Tag push is the right trigger because git-cliff tags are already part of the version SSOT.

## One-time seed

Before CI takes over, fill the gap manually:

1. Install git-cliff locally: `brew install git-cliff`
2. Write `cliff.toml` at repo root
3. Run `git-cliff --bump --unreleased --prepend CHANGELOG.md` to generate the v0.9.19 entry
4. The v0.9.17 placeholder is filled by git-cliff's `--bump --tag v0.9.17` from the existing commits
